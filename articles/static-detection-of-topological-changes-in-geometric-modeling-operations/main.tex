\begin{abstract}
  L'utilisation des opérations géométriques proposées dans les API des modeleurs
  entraîne dans la plupart des cas des changements importants sur les cellules
  topologiques des objets modélisés (sommets, arêtes, faces, etc.). Or, les
  noyaux géométriques de ces modeleurs doivent pouvoir tracer ces changements et
  fournir un journal de bord précis décrivant l'évolution des cellules
  (création, suppression, scission, fusion, modification) à la fin de chaque
  opération. Pour cela, les calculs sont généralement réalisés de manière
  dynamique après application de l'opération, la détection des changements ne
  pouvant se faire \textit{a priori} qu'une fois l'opération réalisée; mais dans
  ce cas, la détection des changements topologiques peut rapidement devenir très
  coûteuse, à mesure que les modèles en construction deviennent importants. Dans
  cet article, nous proposons une méthode pour détecter les changements
  topologiques de manière \emph{statique}. Nos travaux s'appuient sur une
  définition des opérations utilisant des règles de réécriture de graphes. Une
  analyse syntaxique de ces règles permet alors de calculer, de manière statique
  et avant toute application, les changements qui affecteront les entités
  topologiques. Nous définissons également de manière précise les cas limites
  dans lesquels il sera impossible d'affirmer qu'une entité topologique sera
  scindée et/ou fusionnée et où il sera donc nécessaire de procéder à une
  analyse dynamique du modèle.
\end{abstract}
