\textbf{Level 1} The application \texttt{1-square} has no topological parameter
and, thus, is identically reevaluated. Once again, its node $n5$ creates a
single dart $5$ and the volume face $\orbit{0,1}(5)$ is identically reevaluated.
This is why the orbit level 1 of the reevaluation DAG references dart $5$.

\textbf{Level 2} The second application is an added one. \texttt{ADD1-insert}
does match an edge of the tracked volume face $\orbit{0,1}(5)$ and modifies it,
as deduced from the rule. Therefore, the event level 2 contains
\texttt{MODIFICATION} and the orbit level 2 contains the same dart $5$ and orbit
$\orbit{0,1}$.

\textbf{Level 3} The third application is the extrusion of the square face into a
cube. During its application, dart $5$ is matched by the hook $n_1$
(Fig.~\ref{fig:rule-face-extrusion}). From the evaluation DAG, the tracked dart
is the copy of dart $5$ created by node $n_6$. Applying the rule allows finding
out this dart, numbered $59$.

\textbf{Level 4} Finally, the last application of $PN_3$ is deleted.
Consequently, the modification that occurred during the initial evaluation does
not occur at reevaluation. Therefore, the event level 4 contains
\texttt{NOMODIF} and the orbit level 4 contains the same dart~$59$.

\textbf{Level 1} As seen previously, \texttt{1-square} creates the tracked
orbits traced in the evaluation DAG.

\textbf{Level 2} \texttt{ADD1-insert} matches dart $3$. Since the vertex insertion rule
(Fig.~\ref{fig:rule-insert-vertex}) splits the volume face edge $\orbit{0}(3)$,
its history is also split and there are two concrete names to consider. It
follows that the event level contains two \texttt{SPLIT}, one for each volume
face edge suborbit, and two \texttt{NOMODIF}, one for each dart $3$ and $4$.

\textbf{Level 3} \texttt{2-extrusion} extrudes the square face into a cube. The extrusion rule
(Fig.~\ref{fig:rule-face-extrusion}) matches dart $3$ with its hook $n_1$. It
extrudes the face edge $\orbit{0}(3)$ into a volume face, the dart $\orbit{}(3)$
into a face edge and its node $n_2$ creates the dart $37$ as a copy of dart $3$.
The same goes for the history on the right. The rule matches dart $4$, it
creates the same two orbits and a dart $52$ as a copy of dart $4$. % Therefore,
both event levels contain two \texttt{CREATION}. Both orbit levels contain
orbits $\orbit{0,1}$ and $\orbit{0}$. %

\textbf{Level 4} \texttt{5-triangulation} triangulates the faces designated by the level above.
Both darts $37$ and $52$ are matched and preserved by the rule.

Upon reevaluation, $PN_7$'s DAG matches to different darts. An option would be to colour only one face (either $\orbit{0,1}(37)$ or $\orbit{0,1}(52)$). %
Another option would be to apply \texttt{8-colour} two times, one for dart $37$ and one for dart $52$ (as shown in Fig.~\ref{fig:8-reeval} which represent our default strategy). % 
In case the reevaluation DAG has two or more leaves, it shows all the possible entities that can be matched for a specific persistent name and to set a strategy up.
Either way, such a strategy allows users to have a choice and best fit their modeling intents, depending on the application's context.
