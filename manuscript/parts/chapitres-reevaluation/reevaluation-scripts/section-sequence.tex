\section{Structure séquence}
\label{sec:extension-scripts-sequences}



La représentation boîte ouverte pour les scripts nous permet d'accéder à leurs
contenus. %
Nous pouvons alors enregistrer les opérations évaluées au cours de l'application
d'un script ainsi que leurs paramètres topologiques. %
Cette démarche s'inscrit dans la même logique que l'enregistrement des règles. %
En effet, l'affichage du contenu d'un script donne aussi accès au contenu des
règles, c'est-à-dire aux motifs d'orbites. %
Ce dernier est particulièrement important pour tracer les évolutions
d'orbites. %

Dans cette section, nous proposons les premières modifications nécessaires aux
spécifications paramétriques ainsi qu'aux DAGs afin de supporter la réévaluation
des scripts. %
Pour cela, nous prenons l'exemple d'un script extrude un chemin à partir d'un
brin \todo{figure du squelette}. %



\subsection{Spécification paramétrique}
\label{sec:reevaluation-scripts-sequence-spec-param}


\begin{lstlisting}[style=script-style,
language=jerboa-script,
escapechar=\%,
caption=Script de séquence d'extrusions,
label=lst:reevaluation-scripts-sequence-exemple
]
JerboaRuleResult arete1 = @rule<ExtrudeIndependentVertex>(brin#0);
JerboaRuleResult arete2 = @rule<ExtrudeEdgeVertexYAxis>(brin#0@0);
JerboaRuleResult arete3 = @rule<ExtrudeEdgeVertexYAxis>(brin#0@0@1@0);
return null;
\end{lstlisting}
% (liste~\ref{lst:reevaluation-scripts-sequence-exemple})

Prenons le script (liste~\ref{lst:reevaluation-scripts-sequence-exemple}) qui
construit le chemin illustré dans la figure~\todo{}. %
Ce script contient trois appels de règles, chacune opérant l'extrusion d'un
brin. %
La première règle \texttt{Extrude\-Independent\-Vertex} fait l'extrusion d'un brin
libre, c'est-à-dire qui n'est connecté à aucun autre brin. %
Les deux autres règles \texttt{Extrude\-Edge\-Vertex\-YAxis} font l'extrusion d'un
brin à l'extrémité d'une arête. %

\begin{lstlisting}[style=script-style,
escapechar=\%,
caption=Spécification paramétrique avec séquence,
label=lst:reevaluation-scripts-sequence-spec-param]
1-creationBrin()
2-extrusionSquelette(PN%\textsubscript{0}%)
  Sequence:
    1-extrusionSommet(PN%\textsubscript{0}%)
    2-extrusionSommetArete(PN%\textsubscript{1}%)
    3-extrusionSommetArete(PN%\textsubscript{5}%)
3-dualFace(PN%\textsubscript{9}%)
\end{lstlisting}

Détaillons la spécification de la
liste~\ref{lst:reevaluation-scripts-sequence-spec-param}. %
L'application \texttt{1\allowbreak{}-\allowbreak{}creation\-Brin()} crée le brin
\(0\). %
L'application \texttt{2\allowbreak{}\allowbreak{}-S\allowbreak{}%
	\_\allowbreak{}extrusion\-Squelette(PN\textsubscript{0})} entraîne les
applications successives de trois règles d'extrusions à partir du brin \(0\). %
Intuitivement, un script peut être vu comme une spécification paramétrique à
part entière avec une séquence de règles et leurs paramètres. %
En conséquence, un bloc \texttt{Sequence} est déclaré dans la spécification
paramétrique. %
De même, une nouvelle numérotation des opérations est amorcée dans ce bloc. %
Ainsi, à l'intérieur du script, l'application
\texttt{1\allowbreak{}-\allowbreak{}extrusion\-Sommet(PN\textsubscript{0})} crée
une arête à partir d'un brin (\(\orbit{}\)) ainsi que le brin \(1\). %
l'application \texttt{2\allowbreak{}-\allowbreak{}extrusion\-Sommet\-Arete(%
  PN\textsubscript{0,2,3})} extrude une arête à partir du sommet de l'arête
identifiée par le brin \(1\) identifié par le nom persistant \([1n0;2{1n1}]\). %
L'application suivante fait de même à partir du brin \(5\) identifié par
\([1n0;2{1n1;2n2}]\). %
Enfin, dans le bloc principal, l'application
\texttt{3\allowbreak{}-\allowbreak{}dual\-Face} crée une face centrée sur le
sommet identifié par le brin \(9\) identifié par \([1n0;2{1n1;2n3;3n3}]\). %



\subsection{DAGs}
\label{sec:reevaluation-scripts-sequence-extension-dags}



\begin{figure}[htb!]
	\centering
	\includegraphics[scale=.47]{tmp-evaluationTree-pn9}

L'ouverture des scripts dans les spécifications paramétriques donnent la
possibilité d'accéder à leurs opérations internes dans les DAGs également. %
Prenons le DAGs d'évaluation du PN\textsubscript{9} illustré dans la
figure~\ref{fig:reevaluation-scripts-sequence-extension-dags-eval-pn9}. %
Nous pouvons observer deux nouvelles propriétés dans ce DAG. %
Tout d'abord, une boîte englobe les applications appartenant au script
\texttt{S\_extrusionSquelette}. %
Ensuite, et puisque ce script a son propre paramètre topologique, il est
nécessaire de remonter l'historique de ce paramètre. %
Cette branche permet d'identifier le paramètre topologique du script lors de la
réévaluation et poursuivre ou interrompre l'appariement selon l'édition. %

\todo{dag reeval - annuler 1ère application}

Dans cet exemple, l'entrée du script est la même que l'entrée de la règle
\texttt{extrusionSommet}. %
Toutefois, nous pouvons imaginer une configuration où ces deux paramètres sont
différents. %
Ainsi, l'existence de l'une des deux orbites ne garantie pas l'existence de
l'autre au cours de la réévaluation. %
