\section{Main concepts}

In this section, we present the generalized maps, graph transformation rules and their subsequent concepts which are necessary to the understanding of our contribution.

\emph{Generalized maps} (or G-maps) \cite{lienhardt1991topological,
  Damiand-Lienhardt14} allow the representation of manifold geometric objects
(with or without boundaries), based on some cellular $n$-dimensional topological
structure. %
The representation of an object as a G-map comes intuitively from its
decomposition into topological cells (vertices, edges, faces, volumes, and so
on). %
For example, the 3D topological object (Fig.~\ref{fig:decomp-no-decomp}) can be
decomposed into a $3$-dimensional G-map (or $3$-G-map). %
The object is first decomposed into volumes (Fig.~\ref{fig:decomp-volume}). %
These volumes are \emph{linked} along their common faces with an $3$-link, drawn
in green. %
The index "$3$" means that the link connect two $3$-dimensional (possibly a
single one) volumes. %
In the same way, volumes are split into faces connected with blue $2$-links
(Fig.~\ref{fig:decomp-face}). %
Then, faces are split into edges connected with red $1$-links
(Fig.~\ref{fig:decomp-edge}). %
Lastly, edges themselves are split into vertices with black $0$-links
(Fig.~\ref{fig:decomp-vertex}) to produce the $3$-G-map describing the objects
shown in Fig.~\ref{fig:decomp-no-decomp}. %
A G-map is therefore a graph, the nodes of which are the vertices (named
\emph{darts}) obtained at the end of the process and the edges are $i$-links. %

G-maps have conditions guaranteeing objects consistency, for example, two faces
are always linked along an edge.

Topological cells are not explicitly represented in G-maps but only implicitly
defined as subgraphs. %
They can be computed using graph traversals defined by an originating node and
by a given set of link labels named \emph{orbit}. %
For example, the 0-cell adjacent to some dart $a$ (or the object's face matching
$a$) (Fig.~\ref{fig:orbit-3D-vertex-123}) is the sub-graph which contains $a$
and all darts reachable from $a$, using links labelled by $1$, $2$ or $3$ and
the links themselves. %
% (\textit{i.e.} darts $a$, $b$, $c$, $d$, $e$, $f$, $g$ and $h$) and the links
% themselves. %
This sub-graph is denoted by $G\orbit{1,2,3}(a)$ where $\orbit{1,2,3}$ is the
\emph{type} of the orbit and models a vertex. %
% the face $BCED$ (Fig.~\ref{fig:house-ABCDE}). %
The $1$-cell adjacent to $a$ (or the object's edge matching $a$)
(Fig.~\ref{fig:orbit-3D-edge-023}) is the sub-graph $G\orbit{0,2,3}(a)$ which
contains $a$ and all the reachable darts using links labelled by $0$, $2$ or
$3$ % (\textit{i.e.} darts $a, b, l$ and $m$) %
and the corresponding links. %
% It represents the topological edge $BC$. %
% Thanks to $2$-loops on the object border, $G\orbit{0, 2}(c)$
% (Fig.~\ref{fig:house-1-cell-c}) also represents the edge $CE$ matching dart
% $c$. %
The $2$-cell adjacent to $a$ (Fig.~\ref{fig:orbit-3D-face-013}) is the sub-graph
$G\orbit{0,1,3}(a)$. %
The $3$-cell adjacent to $a$ (Fig.~\ref{fig:orbit-3D-volume-012}) is the
sub-graph $G\orbit{0,1,2}(a)$. %
% The $0$-cell adjacent to $a$ (or the object vertex matching dart $a$)
% (Fig.~\ref{fig:house-0-cell-a}) is the sub-graph $G\orbit{1, 2}(a)$ and
% represents vertex $B$. %
Note that orbits are more general than cells. %
For example, the volume edge $G\orbit{0,2}(a)$ (Fig.~\ref{fig:orbit-3D-edge-02})
is an $\orbit{0,2}$ orbit incident to $a$.


\subsection{Graph transformation rules}
\label{ssec:rules}

Jerboas's \cite{belhaouari2014jerboa} graph transformation rules allow the
formalisation of operations over G-maps. %

In a few words, a \emph{rule} $r : L \longrightarrow R$ and a \emph{match}
$m : L \to G$ to a G-map $G$, describe the transformation
$G \longrightarrow^{r,m} H$ from $G$ to $H$. %
The match $m$ allows the replacement of a sub-graph of $G$ described by the
left-hand side of the rule $L$ with another one described by the right-hand side
$R$, in order to produce $H$. %

Informally, in the extrusion rule (Fig.~\ref{fig:rule-face-extrusion}), the
left-hand side is made of only one node $n_{1}$ labelled with the face
$\orbit{0,1}$ type: this way, it can match any face. %
%
For the match $m : n1 \mapsto 6$ from $L$ to $G$
(Fig.~\ref{fig:demo-fig-extrusion-left-match}), the whole face
$G\orbit{0, 1}(6)$ is matched. %
%
On the right side, the node $n_{1}$'s label is $\orbit{0,1}$. %
%
This means that when applying this rule, node $n_{1}$'s $0$- and $1$-links are
preserved and so is the matched face in $H$
(Fig.~\ref{fig:before-after-extrusion}). %
In $R$, the new node $n_{2}$ creates, in $H$, $8$ new darts by copying the $8$
matched dart of $G$. %
However, $n_{2}$'s label is $\orbit{0, \_}$ meaning that $0$-links between the
copied darts are still $0$-links and $1$-links are deleted. %
Therefore $n_{2}$ creates face edges $\orbit{0}$ from the matched face. %
In a similar way, $n_{3}$'s label is $\orbit{\_, 2}$ and, since $0$-links are
deleted and $1$-links relabelled with $2$, creates edge vertices $\orbit{2}$. %
Finally, the nodes $n_{4}$, $n_{5}$, $n_{6}$ create the same orbits than nodes
$n_{3}$, $n_{2}$ and $n_{1}$ respectively. %

All the highlighted links (Fig.~\ref{fig:demo-fig-extrusion-right-match}) are
referred to as \emph{implicit links} in the rule nodes. %
Conversely, the links connecting the nodes together in $R$ are referred as
\emph{explicit links}. %
For example, the explicit $2$-link between nodes $n_{1}$ and $n_{2}$ link
one-to-one the orange and blue darts. %
Therefore, the rule creates the five new faces $\orbit{0, 1}$ for the extrusion
of the matched face into a cube. %

The $n_{1}$ node (Fig.~\ref{fig:rule-face-extrusion}) is a \emph{preserved node}
because it belongs to both the left and right side of the rule. %
Nodes $n_{2}$ to $n_{6}$ are \emph{created nodes} because they belong only to
the right side of the rule. %
This rule does not comprise any \emph{deleted node} because none of its nodes
belong only to the left side. %

For example, in the right-hand side of the triangulation rule
(Fig.~\ref{fig:rule-triangulation}), the $\orbit{0, 2}$-orbit (an edge) incident
to node $n_{0}$ contains the single node $n_0$, and the $\orbit{0, 2}$-orbit
incident to $n_1$ contain the two nodes $n_1$ and $n_2$ and the explicit
$0$-link which connects them. %
Additionally, an $\orbit{o}$-orbit is said to be \emph{complete} if each node in
the orbit matches one link per label of $\orbit{o}$ either explicitly or
implicitly. %
For example, the $\orbit{0, 2}$-orbit incident to node $n_{0}$ in the right-hand
side of the triangulation rule (Fig.~\ref{fig:rule-triangulation}) is not
complete, because node $n_0$ has no $2$-link, either implicitly or explicitly. %
Conversely, the $\orbit{0, 2}$-orbit incident to $n_1$ is complete, because its
two nodes $n_0$ and $n_1$ are incident to the explicit $0$-link and both have an
implicit $2$-link. %
Note that an $\orbit{o}$-orbit in a graph is entirely matched by a rule pattern
if and only if the corresponding $\orbit{o}$-orbit in the pattern is complete. %
For example, the $\orbit{0, 2}$-orbit incident to $a$ in the graph $I$
(Fig.~\ref{fig:demo-fig-triangulation-right-match}) is partially matched by the
triangulation rule (only dart $a_{0}$ and its $0$-linked neighbour are matched),
because the $\orbit{0, 2}$-orbit incident to $n_0$ in the rule is not
complete. %
Conversely, the $\orbit{0, 2}$-orbit incident to $a_1$ in graph $I$ is entirely
matched by the triangulation rule (both darts $a_1$ and $h_1$ are matched by
node $n_1$ and darts $a_2$ and $h_2$ are matched by node $n_2$), because the
$\orbit{0, 2}$-orbit incident to $n_1$ in the rule is complete. %

Jerboa's rules provide syntactic properties guaranteeing G-maps consistency
preservation. %

\subsection{Orbit tracking}

Rules contains the necessary information characterizing the topological changes
affecting an orbit throughout an application. %
Thus, the tracking of an orbit is made automatic without any addition other than
the rules' syntactic analysis. %

For example, the application of the face triangulation rule
(Fig.~\ref{fig:rule-triangulation}) on the front left face of graph $H$
(Fig.~\ref{fig:demo-fig-triangulation-left-match}), produces graph $I$
(Fig.~\ref{fig:demo-fig-triangulation-right-match}). %
The syntactic analysis of the rule allows us to deduce that the vertex
$\orbit{1,2,3}$ orbit (red) incident to the node $n_2$ is \emph{created} because
$n_2$ itself is a created node. %
Moreover, this created vertex comes from the matched face $\orbit{0,1,3}$ orbit
incident to the node $n_0$ in the left-hand side of the rule, because the
implicit links $1,2,3$ of $n_2$ are relabelled from the matched implicit links
$0,1,3$ of $n_0$. %
Consequently, the corresponding orbit (red) in the graph $I$ is created with the
matched face as its \emph{origin}. %

Highlighted in green, the face $\orbit{0,1,3}$ orbit incident to the preserved
node $n_0$ is the whole pattern in the left-hand side of the rule and is the
pattern minus $2$-links (green) in the right-hand side. %
That is, the face orbit is \emph{split} along $1$-links because the second
implicit $1$-link of $n_0$ in the left-hand side is relabelled outside of the
face type orbit $\orbit{0, 1, 3}$ in the right-hand side. %
Consequently, the matched face of graph $H$ is split along its vertices'
$1$-links and into four faces in graph $I$. %
A similar analysis is able to detect when a rule merges two or more orbits of
type $\orbit{o}$ due to a $k$-th implicit link being rewritten within
$\orbit{o}$ in the right-hand side while there was no such $k$-th implicit link
in any left-hand side's node. %

In blue, the face vertex $\orbit{1,2}$ orbit incident to the preserved node
$n_{0}$ is \emph{modified}. %
That is, the vertex orbit incident to $n_{0}$ in the left-hand side matches
$1$-links and because in the right-hand side the orbit comprises the node
$n_{1}$, the orbit comprises $2$-links. %
Since these $2$-links are relabelled from $n_0$'s $1$-links and there is no
split. %
Consequently, the matched vertices of graph $H$ are modified in graph $I$. %

Finally, the face edge orbit $\orbit{0}$ incident to the preserved node $n_{0}$
is \emph{not modified}. %
Since the rule does not rewrite any implicit $0$-link nor adds an explicit
$0$-link to $n_{0}$, the face edge's orbit is not modified and, therefore, the
matched edges from graph~$H$ are not modified in graph~$I$. %

With such analysis, topological changes can be logged within bulletin boards
which are automatically computed without requiring any other
intervention.
