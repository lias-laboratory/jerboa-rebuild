\subsection{title}
\label{sec:orgb421782}
Today, I present a method to automatically detect the topological changes that
modelling operations cause on objects.

A topological change is an event creating, splitting or merging a topological
cell (faces, edges, vertices) within an object.

\subsection{slide 1}
\label{sec:orgc4168e1}
\subsubsection{animation 1}
\label{sec:orga868f42}
For example, take the following object

\subsubsection{animation 2}
\label{sec:orgd149171}
and subdivide it.

\subsubsection{animation 3}
\label{sec:org6266d2f}
Faces are splitted, vertices are created and so on.

Typically, these changes are dynamically detected, that is, by going through the
whole mesh.

After the actual application of an operation.

In addition, the more the object is complex the more the event detection costs.

\subsubsection{animation 4}
\label{sec:org8ff372e}
Nowadays, topological events are tracked in every system, for example when
maintaining the coherence of a model during its conception.

\subsection{slide 2}

To answer these issues, we propose to formalize the static detection of local
events and automate this process based on an automatic analysis of operations.

\subsection{Jerboa's graph transformation rules}
\label{sec:org1b9638a}
\subsubsection{animation 1}
\label{sec:orgeb13211}

\subsubsection{animation 2}
\label{sec:orgfca3484}
A rule has two members. The left member matches a sub-graph out of a G-map. The
right member transforms the former sub-graph into a new one.
Each member contains nodes to match sub-graphs.
\subsubsection{animation 3}
\label{sec:org492aa11}
Node n0, on the left, has labels 0 and 1. As a hook node, when mapped to a0, it

\subsubsection{animation 4}
\label{sec:orgc1d7c9b}
On the right, n0 is preserved and its only label is 0. Therefore the darts are
preserved, the 0-links too, however, 1-links are deleted.
\subsubsection{animation 5}
\label{sec:orgbe338e5}
For consistency purpose a created node is a copy of the hook node. Here, n1 is
created and is a copy of n0. It creates as many darts as n0 matches.

Due to a rewriting of its label, there is no 0-link between darts of index 1 and
1-links are replaced by 2-links. These green links are implicit links.

There is a 1-link between n0 and n1. It connects n0's darts to n1's darts with
1-links. These magenta links are explicit links.
\subsubsection{animation 6}
\label{sec:orgef36dea}
n2 is a created node, it matches the created darts and links in blue.

Moreover, an orbit is said to be complete when each of its node matches either

\section{Detecting topological events}
\label{sec:org6a11843}

\subsection{Creation}
\label{sec:org5a64a17}
Here, the edge orbit only contains created nodes therefore the orbit is created.

By Jerboa's syntactic conditions a created node matches all the possible
dimensions and a created orbit is necessarily complete.

Therefore, since the edge orbit created in the rule is complete, the edges it
matches in the gmap are created.

When starting a modeler, all rules are pre-processed with such a syntactic
analysis. This allows a static detection of all occurring events at the moment a
rule is applied.

\subsection{Transition}
\label{sec:orgc25b7f6}
Now we consider the split event.

\subsection{Split}
\label{sec:orgf6e73a6}
There are two kind of split : implicit and explicit split.
\subsubsection{implicit}
\label{sec:org0641db1}

In the right-hand side of the rule, the face orbit's second implicit links are
either deleted or rewritten from 1 to 2. Therefore, the orbit is splitted
through implicit 1-links in the rule.

The right-hand side matches exactly four faces because there is no 0- or 1-links
to connected them (otherwise there would be no split).

\subsubsection{explicit}
\label{sec:org9b6cd53}
Now consider the rule that disconnects (or unsews) two face edges by cutting
their 2-links and a new object being a patch of quads.

Here we expect to split the edge incident to a0 by cutting its 2-links

In the right-hand side, the explicit 2-link between n0 and n1 is replaced by
2-loops for each of them. The orbit is split in two and, therefore, the matched
edge in the gmap is also split in two.

\subsubsection{false positive split}
\label{sec:org320e444}
Looking at this rule's application, but this time for the vertex orbit, we may
expect two splits.

In fact, there is a catch. only one 1,2 vertex orbit is splitted while the other
is merely modified. Such case is the limit of the static part of the event
detection. However, rules are precisely localized at the moment of their
application and we only need to dynamically check whether or not darts a0 and a1
still belong to the same orbit.
