\begin{resume}{Modélisation et rejeu basés sur des règles}
	Les travaux présentés dans cette thèse s'inscrivent dans le domaine de la
	conception assistée par ordinateur (CAO). La construction d'un objet complexe
	est un processus fastidieux, impliquant des cycles successifs d'essais et de
	corrections. Pour alléger la difficulté de la conception, les systèmes de
	modélisation paramétrique actuels proposent une fonctionnalité de rejeu
	permettant de régénérer un objet grâce à l'édition de ses paramètres.
	Cependant, ce processus nécessite soit une détection des changements
	topologiques basée sur une analyse complète de l'objet modifié, ce qui est
	coûteux ; soit ces changements sont codés explicitement dans les opérations,
	ce qui peut être source d'erreurs. Pour répondre à ces limites, nos travaux se
	concentrent sur le développement d'un système de modélisation basé sur des
	règles, permettant de rejouer un processus constructif, notamment avec
	l'ajout, la suppression et le déplacement d'opérations. Nous considérons des
	opérations formalisées par des règles de transformation de graphe Jerboa.

	Notre première contribution est une analyse syntaxique des opérations permettant
	de détecter les changements topologiques (création, scission, fusion, etc.).
	Cette analyse est réalisée de manière statique sur les règles, indépendamment
	des objets auxquels elles sont appliquées. Ainsi, les changements topologiques
	peuvent être détectés et suivis automatiquement. Dans certains cas, lorsqu'une
	règle n'impacte que partiellement les entités topologiques, une analyse
	localisée au sein de l'objet permet de confirmer ou non les changements
	détectés.

	La seconde contribution s'appuie sur notre approche de détection des changements
	topologiques pour proposer un mécanisme de rejeu. Un processus constructif est
	une suite d'opérations appliquées sur des entités topologiques spécifiques.
	Rejouer un processus constructif nécessite de résoudre le problème classique de
	la nomination persistante des entités topologiques dans un modèle géométrique.
	Pour répondre à ce problème, nous proposons une reconstitution de l'historique
	des entités topologiques référencées dans le processus constructif. Chaque
	entité possède une histoire unique qui l'identifie. Grâce à ce mécanisme, nous
	proposons un référencement robuste des entités topologiques lors du rejeu.


	Notre troisième contribution vise à étendre le mécanisme de rejeu proposé aux
	scripts de règles. Bien que les règles soient des outils puissants, elles
	doivent être combinées pour définir des opérations complexes. Le langage Jerboa
	permet de créer de tels scripts à l'aide de structures de contrôle classiques
	telles que les alternatives et les itérations. Nous proposons une extension de
	notre mécanisme de rejeu pour inclure ces structures de contrôle, offrant ainsi
	une plus grande flexibilité dans la modélisation et la gestion des opérations
	complexes.

	L’ensemble des contributions présentées dans ce manuscrit sont implémentées sous
	la forme d’une extension de la plateforme Jerboa.
\end{resume}

	\begin{resume}{Rule-based modelling and reevaluation}
		The works we present in this thesis take place in the Computer Asssisted
		Design's (CAO) field. %
		Designing a complex object is a tedious processing involving repeated
		cycles of trial and error. %
		In order to alleviate such a difficulty in the designing process, current
		parametric modelling systems offer reevaluation mechanisms allowing a user
		to regenerate an object based on the editing of its parameters. %
		However, such a process requires either of processing an entire mesh in
		order to detect topological changes, which is computationally expensive,
		or hard-coding the changes in the modelling operations, which is
		computationally efficient but increases the risk to introduce detection
		errors. %
		To address these limitations, we focus our works on the development of a
		rule-based modelling system allowing users to reevaluate a modelling
		process especially with the addition, the deletion and the displacement of
		operations. %
		We consider operations formalised with Jerboa's graph transformation
		rules.

		Our first contribution is syntactic analysis of the operations allowing
		for the detection of topological changes (creation, split, merge, and so
		on). %
		These analyses are statically performed on rules, independently of the
		object onto which they are being applied. %
		Topological changes, thus, can automatically be detected and tracked. %
		In some cases, when a rule partially impacts a topological entity, then
		a localised analysis performed in the object is sufficient to assert
		whether the event has occurred. %

		Our second contribution makes use of our topological changes detection
		approach in order to offer a reevaluation mechanism. %
		A modelling process is a record of operations sequentially applied on
		specific topological entities. %
		Reevaluating a modelling process first requires solving the
		long-standing problem of persistently naming topological entities in a
		geometric model. %
		In order to solve this problem, we offer to reconstitute the histories
		of topological entities referenced within a modelling process. %
		Each entity can be identified through its history which is unique. %
		With this mechanism, we offer to reference topological entities in a
		robust way throughout reevaluation. %

		Our third contribution aims to extend our reevaluation mechanism to
		enable the use of scripts. %
		Although rules are powerful tools they need to be combined in order to
		define complex operations. %
		The Jerboa language allows for the creation of such scripts with usual
		control structures such as alternatives and iterations. %
		We offer to extend our reevaluation mechanism by including these control
		structure, hence, enabling the user with greater versatility in
		modelling and managing complex operations. %

		Our contributions are implemented as plugins for the Jerboa platform. %
	\end{resume}
