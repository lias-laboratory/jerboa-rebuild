\section{Main concepts}
\subsection{Generalized maps}

\emph{Generalized maps} (or G-maps) \cite{lienhardt1991topological, Damiand-Lienhardt14} allow the representation of geometric object and their
cellular $n$-dimensional topological structure.

The representation of an object as a G-map comes intuitively from its
decomposition into topological cells (vertices, edges, faces, volumes, etc.).
For example, the 2D topological object (Fig.~\ref{fig:house-ABCDE}) can be
decomposed into a $2$-dimensional G-map (or $2$-G-map). The object is first
decomposed into faces (Fig.~\ref{fig:house-faces-decomposition}). These faces
are \emph{linked} along their common edge with an $\alpha 2$ link. The index 2
indicates that two cells of dimension 2 (faces) share an edge. $\alpha 2$ links
are drawn in blue and its the next figures its label will be omitted. In the
same way, faces are split into edges connected with the red $\alpha 1$ links
(Fig~\ref{fig:house-show-1-links}). At last, edges are split into vertices by
the black $\alpha 0$ link to obtain the $2$-G-map
(Fig~\ref{fig:house-show-all-links}). Vertices obtained at the end of the
process are the nodes of the G-map graph also named \emph{darts} and $\alpha i$
links are its edges.

G-maps have conditions guaranteeing objects coherence, for example, two faces
are always linked along an edge.

Topological cells are not explicitly represented in G-maps but only implicitly
defined as subgraphs. They can be computed using graph traversals defined by an
originating node and by a given set of link labels named \emph{orbit}. For
example, the 2-cell adjacent to $a$ (or object face attached to the dart $a$)
(Fig.~\ref{fig:house-2-cell-a}) is the subgraph which contains $a$, darts
reachable from the dart $a$ using links labeled by $\alpha 0$ or $\alpha 1$
(darts $a$, $b$, $c$, $d$, $e$, $f$, $g$ and $h$) and the links themselves. This
subgraph is denoted by $G\orbit{\alpha 0 \alpha 1}(a)$ and models the face
$BCED$ (Fig.~\ref{fig:house-ABCDE}). $\orbit{\alpha 0, \alpha 1}$ is the
\emph{type} of the orbit. The $1$-cell adjacent to $a$ (or object edge attached
to the dart $a$) (Fig.~\ref{fig:house-1-cell-a}) is the subgraph
$G \orbit{\alpha 0 \alpha 2}(a)$ that contains the dart $a$ and all darts that
are reachable by using links labeled by $\alpha 0$ or $\alpha 2$ (darts
$a, b, l$ and $m$) and the corresponding links. It represents the topological
edge $BC$. Thanks to $\alpha 2$-loops on the object border,
$G\orbit{\alpha 0, \alpha 2}(c)$ (Fig.~\ref{fig:house-1-cell-c} also represents
the edge $CE$ attached to dart $c$. The $0$-cell adjacent to $a$ (or object
vertex attached to dart $a$) (Fig.~\ref{fig:house-0-cell-a}) is the subgraph
$G\orbit{\alpha 1, \alpha 2}(a)$ and represents the vertex $B$. Orbits are more
general than cells. For example, the face edge $G\orbit{\alpha 0}(a)$
(Fig.~\ref{fig:house-1-cell-a0-a}) is an $\orbit{\alpha 0}$ orbit incident to
$a$.

\subsection{Graph transformation rules}

Graph transformation rules of Jerboa \cite{belhaouari2014jerboa} allow to
formalize operations of G-maps.

  \caption{Rule $L \longrightarrow R$ of face triangulation}
  \label{fig:rule-triangulation}
\end{figure}

    \caption{Graph $H$}
    \label{fig:house-triangulation-right-match}
  \end{subfigure}
  \caption{Application of the triangulation rule of figure \ref{fig:rule-triangulation} on matched dart $a_{0}$}
  \label{fig:before-after-house-triangulation}
\end{figure}

In a few words, a \emph{rule} $r : L \longrightarrow R$ and a \emph{match}
$m : L \to G$ to a G-map $G$, describe the transformation
$G \longrightarrow^{r,m} H$ of $G$ to $H$. The match $m$ allows to compute a
sub-graph of $G$ from left hand side of the rule $L$ and its replacement from
right hand side $R$ to obtain $H$.

Intuitively, the triangulation rule (Fig.~\ref{fig:rule-triangulation}) in its
left side comprises only one node $n0$ labeled with the face
$\orbit{\alpha 0, \alpha 1}$ type. This way it allows to match any face. For the
match $m : n0 \mapsto a_{0}$ from $L$ to $G$
(Fig.~\ref{fig:house-triangulation-left-match}), the entire face
$G\orbit{\alpha 0, \alpha 1}(a_{0})$ is matched, highlighted in orange in the
figure. In the right side, the node $n0$ is labeled by $\orbit{\alpha 0, \_}$.
The $\alpha 0$ indicates the first links are preserved and "\_" indicates the
second links $\alpha 1$ are deleted. Consequently in $H$
(Fig.~\ref{fig:house-triangulation-right-match}), the $\alpha 0$-links of the
matched orange face are preserved and $\alpha 1$-links are deleted. In other
words the edges of the orange face are disconnected in $H$. The new node $n_{2}$ in
$H$ creates darts copied from the matched face. This is why there are eight blue
darts ($a_2$, $b_2$ \dots $h_2$) created from orange darts ($a_0$, $b_0$ \dots
$h_0$). Because $n_{2}$ is labeled $\orbit{\alpha 1, \alpha 2}$, the first orange
$\alpha 0$ links are relabeled to $\alpha 1$ links in the blue copy. In the same
way, the second orange $\alpha 1$ links are relabeled to $\alpha 2$ links in the
blue copy. Therefore the rule creates a dual vertex to the matched face.
Finally, the node $n_{2}$ creates a green copy with eight darts, deleted the first
$\alpha 0$ links and relabeled the second $\alpha 1$ links to $\alpha 2$ links.

All the highlighted links (Fig.~\ref{fig:before-after-house-triangulation}) are
referred as \emph{implicit links} in the rule nodes. Conversely, the links
connecting the nodes together in $R$ are referred as \emph{explicit links}. For
example, the explicit $\alpha 0$ links between $n_{1}$ and $n_{2}$ link one-to-one the
green and blue darts. Therefore the rule creates the four new edges
$\orbit{\alpha 0, \alpha 2}$ in the triangulation of the matched square face. In
the same way, the explicit $\alpha 1$ links between $n_{0}$ and $n_{1}$ link
one-to-one the orange and green darts.

The $n0$ node (Fig.~\ref{fig:rule-triangulation}) is a \emph{preserved node}
because it belongs to both the left and right side of the rule. Nodes $n_{1}$ and
$n_{2}$ are \emph{created nodes} because they belong only to the right side of the
rule. This rule does not comprise any \emph{deleted node} because none of its
nodes belong only to the left side.

\begin{definition}[Complete orbit]
\label{def:complete-orbit}
  Let $r : L \longrightarrow R$ be a rule, $n$ a node such that $n \in L$ and
  $\orbit{o}$ an orbit type.

  $L\orbit{o}(n)$ is a \emph{complete orbit} on $\orbit{o}$ if and only if for
  all nodes $n'\in L\orbit{o}(n)$, $n'$ filters either implicitly or explicitly
  all dimensions in $\orbit{o}$, i.e. for all dimensions $i \in \orbit{o}$, $n'$
  is either the source of an $\alpha i$-link or $i \in l(n')$.

\end{definition}

\section{Event detection}

Intuitively, to detect that a rule application creates (or splits, merges, …) an orbit, one would compare the graph before and after its application.
However, comparing the members of a rule against each other allows a static detection of topological events.
Such a method presents the advantage of being independent of the object transformed by the rule.\end{document}
