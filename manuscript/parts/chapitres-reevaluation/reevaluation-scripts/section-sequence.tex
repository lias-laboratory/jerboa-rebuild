\section{Structure séquence}
\label{sec:extension-scripts-sequences}



La représentation boîte ouverte pour les scripts nous permet d'accéder à leurs
contenus. %
Nous pouvons alors enregistrer les opérations évaluées au cours de l'application
d'un script ainsi que leurs paramètres topologiques. %
Cette démarche s'inscrit dans la même logique que l'enregistrement des règles. %
En effet, l'affichage du contenu d'un script donne aussi accès au contenu des
règles, c'est-à-dire aux motifs d'orbites. %
Ce dernier est particulièrement important pour tracer les évolutions
d'orbites. %

Dans cette section, nous proposons les premières modifications nécessaires aux
spécifications paramétriques ainsi qu'aux DAGs afin de supporter la réévaluation
des scripts. %
Pour cela, nous prenons l'exemple d'un script extrude un chemin à partir d'un
brin \todo{figure}. %



\subsection{Spécification paramétrique}
\label{sec:reevaluation-scripts-sequence-spec-param}


\begin{lstlisting}[style=script-style,
language=jerboa-script,
escapechar=\%,
caption=Script de séquence d'extrusions,
label=lst:reevaluation-scripts-sequence-exemple
]
JerboaRuleResult arete1 = @rule<ExtrudeIndependentVertex>(brin#0);
JerboaRuleResult arete2 = @rule<ExtrudeEdgeVertexYAxis>(brin#0@0);
JerboaRuleResult arete3 = @rule<ExtrudeEdgeVertexYAxis>(brin#0@0@1@0);
return null;
\end{lstlisting}
% (liste~\ref{lst:reevaluation-scripts-sequence-exemple})

Prenons le script (liste~\ref{lst:reevaluation-scripts-sequence-exemple}) qui
construit le chemin illustré dans la figure~\todo{}. %
Ce script contient trois appels de règles, chacune opérant l'extrusion d'un
brin. %
La première règle \texttt{Extrude\-Independent\-Vertex} fait l'extrusion d'un brin
libre, c'est-à-dire qui n'est connecté à aucun autre brin. %
Les deux autres règles \texttt{Extrude\-Edge\-Vertex\-YAxis} font l'extrusion d'un
brin à l'extrémité d'une arête. %

\begin{lstlisting}[style=script-style,
escapechar=\%,
caption=Spécification paramétrique avec séquence,
label=lst:reevaluation-scripts-sequence-spec-param]
1-creationBrin()
2-extrusionSquelette(pn1)
  Sequence:
    1-extrusionSommet(pn2)
    2-extrusionSommetArete(pn3)
    3-extrusionSommetArete(pn4)
\end{lstlisting}

L'application de ce script sur un brin libre donne la spécification paramétrique
représentée par la liste~\ref{lst:reevaluation-scripts-sequence-spec-param}. %
Intuitivement, une spécification paramétrique peut être vue comme un script qui
contient une séquence de règles avec des paramètres et inversement. %
Ainsi, dans nos spécifications paramétriques, nous proposons de représenter les
scripts comme des spécifications imbriquées. %
Dans la liste~\ref{lst:reevaluation-scripts-sequence-spec-param}, la deuxième
opération enregistrée est un script. %
En conséquence, nous ouvrons un bloc \texttt{Sequence} dans lequel nous
enregistrons les opérations de la séquence qui sont évaluées. %
Toute comme dans une spécification paramétrique, les opérations d'un script
disposent d'un identifiant «~frais~» et d'un ensemble de paramètres. %
Dans notre exemple, nous obtenons alors la séquence de règles
\texttt{1\allowbreak{}-\allowbreak{}extrusion\-Sommet(PN\textsubscript{2})},
\texttt{2\allowbreak{}-\allowbreak{}extrusion\-Sommet\-Arete(PN\textsubscript{3})}
et
\texttt{3\allowbreak{}-\allowbreak{}extrusion\-Sommet\-Arete(PN\textsubscript{4})}. %

\subsection{DAGs}
\label{sec:reevaluation-scripts-sequence-extension-dags}

\todo{DAG évaluation avec bloc boîte ouverte}

L'utilisation d'une boîte ouverte pour représenter les opérations internes à un
script nous permet de reconstituer les historiques d'évolutions de chacun des
paramètres topologiques dans des DAGs. %
Afin de faire la distinction entre les opérations hors scripts et les opérations
dans les scripts, le modèle de boîte ouverte doit être reportée dans les DAGs
notamment parce que, tout comme les règles, les scripts peuvent référencer une
entité topologique comme paramètre. %
Ce paramètre conditionne alors l'exécution du script et l'ouverture de la
boîte. %
