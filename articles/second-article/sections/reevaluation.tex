\section{Reevaluation}

An evaluation DAG offer to represent the history of a topological entity. %
However, when a parametric specification is edited for re-evaluation, the
history may also be edited. %
A matching mechanism must be implemented in order to match the expected
topological entities despite the parametric specification editing. %
Therefore, a re-evaluation DAG is built for each evaluation DAG, thus allowing
the matching of the topological entities in the new construction. %
