\section{Scripts}
\label{sec:extension-scripts-scripts}

Les règles Jerboa nous permettent de représenter diverses opérations de
modélisations plus ou moins complexes. %
Toutefois, comme nous l'avons vu dans le
chapitre~\ref{cha:outils-modelisation-base-regles} avec la
définition~\ref{def:orbite-instanciation}
(page~\pageref{def:orbite-instanciation}), les orbites filtrées par la partie
gauche d'une règle doivent être isomorphes à renommage près. %
Ainsi, un langage de script~\cite{belhaouari2014jerboa,gauth2019} a été conçu
afin de permettre l'écriture d'opérations de modélisations plus complexes,
c'est-à-dire des opérations qui peuvent être appliqués sur des orbites non
isomorphes. %

Les scripts permettent d'organiser des règles de transformation de graphe et
ainsi construire des opérations complexes. %
Dans cette section, nous présentons d'abord les scripts proposés par le langage
Jerboa\cite{belhaouari2014jerboa,gauth2019}, puis nous définissons les scripts
que nous proposons à partir d'un sous-ensemble de fonctionnalités proposées par
les scripts Jerboa. %
Descriptions des scripts Jerboa~\cite{belhaouari2014jerboa,gauth2019}

\begin{itemize}
	\item opérer sur plusieurs orbites non isomorphes
	\item paramètres topologiques et de plongements en entrées,
	\item langage de calcul
\end{itemize}
de leurs
structures et de leurs fonctionnalités~: entrée/sortie comme une règle, code
pour organiser des règles jerboa ou manipulation directe de G-carte, utilisation
des structures de contrôles existantes en Java et en C++ (séquence, for, foreach, while,
if then else, etc.)
