
\section{Gestion des iterations sur les sous-orbites}
\label{sec:extension-scripts-iteration}

\begin{lstlisting}[style=script-style,
language=jerboa-script,
escapechar=\%,
caption=Script de séquence d'extrusions,
label=lst:reevaluation-scripts-sequence-exemple
]
for(JerboaDart vertex : <0,1,2>_<1,2>(lin_skeleton#0)){
    float offset = Vec3.dist(vertex.pos, vertex@0.pos);
    Vec3 center = new Vec3(vertex.pos);
    JerboaRuleResult squareFace = @rule<VertexDualToFace>(center, offset/2);
    JerboaRuleResult cube = @rule<ExtrudeIndependentFaceWithSide>(squareFace#0#0, offset);
}
return null;
\end{lstlisting}

Les orbites ont la propriété de pouvoir être décomposées en sous-orbites,
c'est-à-dire, par exemple, que l'on peut extraire les sommets \(\orbit{1}\)
d'une face \(\orbit{0,1}\). %
Une des structures de contrôle proposées par le langage Jerboa et l'itération
des sous-orbites d'une orbite. %
Un exemple d'une telle itération peut-être le script qui, pour chaque sommet
d'une face, applique la règle \texttt{chanfrein\-Sommet}. %

De la même manière que pour les séquences d'opérations, il est nécessaire
d'étendre la définition d'une spécification paramétrique. %
Cette extension consiste à permettre la représentation des itérations de cette
structure de contrôle. %
Ainsi, une spécification paramétrique contiendra, dans la boîte de script, son
propre bloc d'opérations. %
Un bloc d'itération dispose alors d'un nom persistant comme paramètre d'entrée
qui conditionne sa réévaluation et les différentes applications liées aux
sous-orbites de l'itération. %



\subsection{Extension des DAGs}
\label{sec:reevaluation-scripts-iteration-extension-dags}

\todo{dag évaluation boîte ouverte + bloc itération (pn) + séquences d'itérations (pn)}

En plus des orbites référencées par les scripts, afin de supporter les
itérations sur les sous-orbites, il est désormais nécessaire de suivre d'autres
orbites pour conditionner la réévaluation d'un script. %
Plus précisément, il convient de tracer l'historique d'évolution de l'orbite
passée en paramètre de la boucle ainsi que les historiques des sous-orbites
itérées. %
Dans chaque DAG illustrant l'historique d'une sous-orbite itérée, il y a
désormais une branche qui trace l'historique du paramètre topologique du script,
une branche qui trace l'historique de l'orbite passée en paramètre de la
structure de contrôle et une branche qui trace l'historique pour une instance de
sous-orbite. %
