Contrary to its evaluation DAG, a reevaluation DAG is built from start to end \david{top-down ?} throughout the reevaluation process. %
It describes the history of an orbit in the context of the new modelling process after edition. %
While an evaluation DAG represents any possible topological parameter \david{je ne comprends pas trop ce "any possible". Pourquoi pas simplement "a" ?} through the set of darts matched by rule nodes of the DAG,
the reevaluation DAG represents the new designated topological parameters through the chosen darts. %

For example, let's consider $PN_{3}$ being the topological parameter of \texttt{4-extrusion} which extrudes the cube's top face
to produce %into
a second cube. %
Its reevaluation DAG (Fig.~\ref{fig:pn3-reval-dag}) is built step-by-step during the reevaluation process from its evaluation DAG (Fig.~\ref{fig:eval-dag-pn3}). %
\begin{description}
  \item[Level 1] The application \texttt{1-square} has no topological parameter and, thus, is identically reevaluated. Once again, its node $n5$ creates a single dart $5$ and the volume face $\orbit{0,1}(5)$ is identically reevaluated. This is why the orbit level 1 of the reevaluation DAG  references dart $5$.
  \item[Level 2] The second application is an added one. Therefore, this level only depends on the tracking of the orbit through this new modelling step. \texttt{ADD1-insert} does match an edge of the tracked volume face $\orbit{0,1}(5)$ and modifies it. Therefore, the event level 2 contains \texttt{MODIFICATION} and the orbit level 2 contains the same dart $5$ and orbit $\orbit{0,1}$.
  \item[Level 3] The third application is the extrusion of the square face into a cube. During its application, dart $5$ is matched by the hook $n1$. From the evaluation DAG, the tracked dart is the copy of dart $5$ created by node $n6$.
  The rule's application allows finding out this dart,
  numbered %is
  $59$.
  \item[Level 4] Finally, the last application of $PN_3$ is deleted. Consequently, the modification that occurred during the initial evaluation does not occur at reevaluation. Therefore, the event level 4 contains \texttt{NOMODIF} and the orbit level 4 contains the same dart~$59$.
\end{description}
The reevaluation DAG designates the concrete name from its persistent name.

\subsection{Parameter matching strategies}

A topological parameter can be duplicated by an added operation %splitting an orbit. %
such as an orbit split. %
%A strategy could be that,
One could set up a strategy such that,
during the reevaluation, the application should be applied on as many parameters as there are suborbits resulting from an orbit being split.

\begin{figure}[ht]
    \centering
    \includegraphics[width=\columnwidth]{pn7-reval}
    \caption{$PN_7$ reevaluation DAG}
    \label{fig:pn7-reval-dag}
\end{figure}

For example, let's consider the topological parameter for \texttt{8-colour}, $PN_7$, being the left face resulting from the triangulation (Fig.~\ref{fig:8-eval}). %
$PN_7$'s reevaluation DAG is shown in Fig.~\ref{fig:pn7-reval-dag}.%
\begin{description}
  \item[Level 1] The first application which creates a square face is reevaluated identically and creates a face edge $\orbit{0}$ and a dart orbit $\orbit{}$. %
  The node $n_3$ creates a single dart $3$. The related event level contains two \texttt{CREATION}; the orbit level references dart $3$ and contains two orbits $\orbit{0}$ and $\orbit{}$.
  \item[Level 2] The added application \texttt{ADD1-insert} matches dart $3$. Since the rule splits the face edge $\orbit{0}(3)$, its history is also split and there are two concrete names to consider. It follows that the event level contains two \texttt{SPLIT}, one for each face edge suborbit, and two \texttt{NOMODIF}, one for each dart.
  The left orbit level references dart $3$ and contains two orbit $\orbit{0}$ and $\orbit{}$. The right orbit level references dart $4$ and contains the same two orbits $\orbit{0}$ and $\orbit{}$.
  \item[Level 3] The third application extrudes the square face into a cube. The rule's hook $n1$ matches dart $3$. It extrudes the face edge $\orbit{0}(3)$ into a volume face, the dart $\orbit{}(3)$ into a face edge and its node $n2$ creates the dart $37$ as a copy of dart $3$. The same goes for the history on the right. The rule matches dart $4$, it creates the same two orbits and a dart $52$ as a copy of dart $4$. %
  Therefore, both event levels contain two \texttt{CREATION}. Both orbit levels contain orbits $\orbit{0,1}$ and $\orbit{0}$. However, the left orbit level references dart $37$ while the right orbit level references dart $52$. %
  \item[Level 4] The last application, \texttt{5-triangulation} triangulates the faces designated by level 3. Since both darts are matched and preserved by the rule, two darts are referenced: dart $37$ on the left orbit level and dart $52$ on the right orbit level. %
\end{description}
