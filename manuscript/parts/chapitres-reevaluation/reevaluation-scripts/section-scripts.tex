Les règles Jerboa nous permettent de représenter diverses opérations de
modélisations plus ou moins complexes. %
Toutefois, comme nous l'avons vu dans le
chapitre~\ref{cha:outils-modelisation-base-regles} avec la
définition~\ref{def:orbite-instanciation}
(page~\pageref{def:orbite-instanciation}), les orbites filtrées par la partie
gauche d'une règle doivent être isomorphes à renommage près. %
Ainsi, un langage de script~\cite{belhaouari2014jerboa,gauth2019} a été conçu
afin de permettre l'écriture d'opérations de modélisations plus complexes,
c'est-à-dire des opérations qui peuvent être appliqués sur des orbites non
isomorphes. %
%% Exemple de problème lié à l'isomorphisme des règles Jerboa
La figure~\ref{fig:reevaluation-scripts-scripts-scripts-jerboa-non-decousure}
illustre un exemple où la décousure d'un volume par
ses 3-liaisons
(figure~\ref{fig:reevaluation-scripts-scripts-scripts-jerboa-regle-decousure})
est impossible, car certaines de ces faces sont libres. %
Leurs 3-liaisons ne peuvent donc pas être découpées et le volume orange ne peut
pas être déconnecté du reste de la composante connexe. %


\subsection{Scripts Jerboa}
\label{sec:extension-scripts-scripts-scripts-jerboa}

Pour dépasser cette limitation des règles,
%% Écriture d'un script, instruction « élémentaire » : appel de règle
Jerboa propose des scripts construits à partir de deux langages. %
Un langage de calcul est d'abord conçu pour définir un ensemble d'instructions,
comprenant entre autres des structures de contrôles, afin réaliser des
opérations sur les G-cartes et leurs orbites. %
Un langage de script étend ensuite ce langage de calcul avec des appels
d'opérations qui en sont les instructions élémentaires. %
Les scripts permettent ainsi d'appeler des règles Jerboa et de les organiser à
l'aide de structures de contrôle afin de concevoir des opérations complexes. %


\begin{lstlisting}[style=script-style,language=jerboa-script]
for(JerboaDart face : <0,1,2>_<0,1>(volume#0))
{
    try
    {
        @rule<UnsewFace01A3>(face);
    } catch (JerboaException e){
        // Ici l'application echoue
    }
}
return null;
\end{lstlisting}
