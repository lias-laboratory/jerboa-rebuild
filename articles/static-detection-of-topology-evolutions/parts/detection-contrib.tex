\section[Event detection]{Event detection}
\label{sec:event-detection}

Intuitively to detect that a rule creates some topology we would compare the
graph before and after a rule's application. On the other hand comparing the
members of a rule against each other allow us to statically detect topological
events. Such method is independent of the object affected by the rule.

\subsection[Creation]{Creation}
\label{sec:creation}

In a rule a node filters one or more darts. So when a node is created the darts
it filters are also created.

\begin{definition}
\label{def:creation}
Let there be a rule $r: L \to R$, a G-map $G$, an orbit type $\orbit{o}$, a
node $n$ where $n \in R$ and an \emph{application}
$G \longrightarrow^{r,m} H$.

If an orbit $R\orbit{o}(n)$ is both \emph{complete} and created by the rule,
i.e. none of the nodes in $R\orbit{o}(n)$ belong to $L$, then and for all darts
$d$ instance of $n$, $d\orbit{o}(n)$ is created in $H$.
\end{definition}

Definition \ref{def:creation} gives the necessary and sufficient conditions
under which we can confirm a rule will create some orbit.

Given the face triangulation rule figure \ref{fig:rule-triangulation} we will
follow the creation event of a vertex of orbit type
$\orbit{\alpha 1, \alpha 2}$. This rule creates two nodes in $R$, namely,
$n_{1}$ and $n_{2}$. Starting with $R\orbit{\alpha 1, \alpha 2}(n_{1})$ we
obtain an orbit containing nodes $n_{0}$ and $n_{1}$. As this orbit contains the
\emph{preserved} node $n_{0}$ we cannot consider the topology it filters as
created. On the contrary the orbit $R\orbit{\alpha 1, \alpha 2}(n_{2})$ only
contains $n_{2}$. This orbit being fully created we can affirm the rule of face
triangulation will create a vertex filtered by node $n_{2}$ during its
application.

Figure \ref{fig:before-after-house-triangulation} shows the application of the
rule \ref{fig:rule-triangulation} on a square face. In figure
\ref{fig:house-triangulation-vertex-creation} the orbit
$R\orbit{\alpha 1, \alpha 2}(n_{2})$ is in a blue highlight (L'orbite concrète
sera donné après mise à jour des figures).

\subsection[Split]{Split}
\label{sec:split}


Given the face triangulation rule figure \ref{fig:rule-triangulation}, the graph
before and after the rule's application in figures
\ref{fig:house-triangulation-left-filter} and
\ref{fig:house-triangulation-split-face} and the orbit type $\orbit{\alpha 0,\alpha1}$ of a
face.

In figure \ref{fig:house-triangulation-left-filter} the darts highlighted in
orange belong to face splitted by the face triangulation rule. We can notice
these darts form the borders of the square face, its edges. In figure
\ref{fig:house-triangulation-split-face} these darts still form the same edges
and are still highlighted in orange. Yet they do not belong to the same face
anymore. Also these new faces being 2-adjacent and connected by a common vertex
we can conclude the former square face has been splitted into 4 triangle faces.

Again to statically detect a split event we only consider the face triangulation
rule. In this case the node of interest is $n_{0}$. In the left member of the
rule this node filters a face as the orbit type $\orbit{\alpha 0,\alpha1}$ suggests. In the
right member the face orbit $n_{0}$·$\orbit{\alpha 0,\alpha1}$ contains the nodes $n_{0}$,
$n_{1}$ and $n_{2}$. We now have two face orbits: the left one
$L\orbit{\alpha 0,\alpha1}(n_{0})$ and the right one $R\orbit{\alpha0,\alpha1}(n_{0})$.
