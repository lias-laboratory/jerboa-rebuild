\section{Nomination persistante des orbites}
\label{sec:reevaluation-nomination-persistante-orbites}

Pour compléter la nomination des entités topologiques, nous réutilisons les noms
persistants de brins pour construire les arbres d'évaluations, ou DAGs d'évaluations, qui
représentent tous les évènements qui ont impacté chacune des orbites, autrement
leurs historiques d'évolutions. %

% \maxime{idée plan~:\\
% 	Combinaison d'orbites d'origine et de suivi dans un DAG pour une représentation
% 	complète d'un évènement à chaque application de règles. Formalisation de l'usage
% 	de chemins pour préciser une origine lorsque celle-ci n'est pas directement liée
% 	à un nœud d'accroche dans une règle. }

% \begin{itemize}
% 	\item les entrées pour la construction du DAG sont~: un nom persistant de
% 	      brin, le type d'orbite de l'entité topologique, une spécification
% 	      paramétrique (pour associer numéro d'opération et opération)
% 	\item les notions de suivi (flèche noire) et d'origine (flèche rouge)
% 	      d'une orbite (au travers d'une opération)
% \end{itemize}

Détaillons la construction du DAG d'évaluation pour le paramètre topologique l'application
\texttt{8-colorationBleu(\pndart{35}{1n3;2n4;5n0}, Bleu)} de la construction nº~1. %

\begin{figure}[htb!]
	\centering
	\includegraphics[height = 40pt]{rule-colour-face-n0-highlight}
	\caption{}
	\label{fig:reevaluation-base-regle-nomination-persistante-orbites-type-orbite}
\end{figure}

La première étape, pour construire le DAG d'évaluation, consiste à identifier le
type d'orbite filtré par la règle. %
Comme expliqué dans les chapitres précédents, cette information est représentée
dans l'étiquette du n\oe ud d'accroche de la règle. %
Pour notre exemple, le type d'orbite qui nous intéresse est l'étiquette du n\oe
ud \(n_{0}\) affiché en {\color{orange}orange} dans la
figure~\ref{fig:reevaluation-base-regle-nomination-persistante-orbites-type-orbite}. %
À partir de cette information, nous pouvons commencer à remonter l'historique
des évolutions de l'orbite sur laquelle l'opération de coloration a été
appliquée. %
Autrement dit, l'historique que nous reconstituons est un DAG qui se construit
et se lit de la fin jusqu'au début, de l'état le plus récent de l'orbite jusqu'à
ses ancêtres les plus anciens. %

\begin{figure}[htb!]
  \centering
  \includegraphics[height=20pt]{pn35-dag-eval-step-0}
  \caption{Racine du DAG d'évaluation}
  \label{fig:reevaluation-base-regle-nomination-persistante-orbites-construction-dag-racine}
\end{figure}

Ainsi, le type de l'orbite que nous suivons, ou type d'orbite de \emph{suivi},
est le premier élément intégré dans le DAG en tant que racine
(figure~\ref{fig:reevaluation-base-regle-nomination-persistante-orbites-construction-dag-racine}). %

\begin{figure}[htb!]
  \centering
  \includegraphics[height=105pt]{pn35-dag-eval-step-2}
  \caption{Étape 2 \texttt{[1n3;2n4;{\color{orange}5n0}]} de la construction du DAG}
  \label{fig:reevaluation-base-regle-nomination-persistante-orbites-construction-dag-etape-2}
\end{figure}

La suite de la reconstitution consiste à remonter l'historique du nom persistant
du brin, application par application. %
Dans le DAG d'évaluation, une application est représentée par un couple de
niveaux~: (1) un niveau d'orbite qui contient l'ensemble des types d'orbites
suivis dans l'application courante~; (2) un niveau d'événements qui ont impacté
les orbites suivies. %

\begin{figure}[htb!]
  \centering
  \includegraphics[height=45pt]{rule-face-triangulation-3D-n0-highlight-boxed-v2}
  \caption{Analyse de la règle de triangulation}
  \label{fig:reevaluation-base-regle-nomination-persistante-orbites-construction-dag-analyse-triangulation}
\end{figure}

Reprenons notre exemple. %
La deuxième étape de la reconstitution d'historique commence avec la dernière
application enregistrée dans le nom persistant du brin \(35\), c'est-à-dire
l'application \texttt{5-triangulation} et son n\oe ud \(n_{0}\). %
Grâce à ces informations, nous pouvons procéder à l'analyse de la règle de
triangulation
(figure~\ref{fig:reevaluation-base-regle-nomination-persistante-orbites-construction-dag-analyse-triangulation})
et plus particulièrement à la face incidente à \(n0\), \(\orbit{0,1}(n_{0})\). %
L'étude de cette orbite, comme nous l'avons vu dans la
section~\ref{sec:formalisation-evenements-regles-transfo-graphe-scission}
(page~\ref{sec:formalisation-evenements-regles-transfo-graphe-scission}), montre
que l'orbite \(L\orbit{0,1}(n_{0})\) est scindée du fait de la réécriture de
tous les \(1\)-arcs implicite dans des dimensions hors du type \(\orbit{0,1}\),
voire de leur suppression. %
De plus, nous pouvons voir dans l'orbite \(R\orbit{0,1}(n_{0})\) que les \(0\)
et \(1\)-arcs implicites sont des réécritures d'un \(0\)-arc implicite dans le
n\oe ud d'accroche. %
Ainsi, l'orbite que nous suivons, d'une part, est scindée, et, d'autre part, a
pour origine une arête de type \(\orbit{0}\). %
